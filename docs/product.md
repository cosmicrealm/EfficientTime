# EfficientTime 产品设计

## 产品定位

EfficientTime 不是通用 Todo，也不是完整日历。它的核心是把已经确定要做的事情转成可执行的当天严格时间表，并在执行过程中持续提醒用户。

## 第一版边界

- 本地单机。
- 最新 macOS。
- 严格时间表。
- 桌面悬浮窗常驻，可紧凑/展开。
- DeepSeek 配置只在设置页出现；AI 只做计划拆分和草稿建议，不直接写入正式计划。
- 支持今天和明天的独立规划。

## 核心模式

### Plan Mode

用户输入可用时间段、固定任务和待安排任务。系统先用本地排程引擎生成严格时间表。AI 可辅助拆分任务、追问约束和建议顺序。

默认添加路径应保持简单：任务名称、开始时间、结束时间。

### Run Mode

用户点击开始后，菜单栏和悬浮窗持续显示当前任务、剩余时间和下一个任务。任务状态可切换为完成、跳过、延长、推迟。

悬浮窗默认显示当前任务附近最多 5 个事项：前 2 个、当前 1 个、后 2 个，并支持折叠成小条。

### Review Mode

当天结束后展示完成率、跳过事项、延迟事项和执行日志。后续 AI 可基于脱敏摘要生成复盘。

## AI 规划原则

AI 的输出是 `PlanDraft`，正式计划必须经过本地校验和用户确认。

默认只发送最小必要上下文：

- 今天日期。
- 可用时间段。
- 脱敏后的任务标题。
- 预计耗时。
- 固定时间约束。
- 优先级。
- 可拆分性。

默认不发送：

- 完整历史执行日志。
- 财务账户细节。
- 私密备注。
- 本地文件内容。
- 联系人、完整日历或其他 App 数据。
